!!! warning "2026 Update — what changed since this was written (as of May 2026)"

    - 2026-standard RAG practice now assumes: agentic/iterative retrieval, hybrid (dense + sparse) search with a reranking stage, and **contextual retrieval** (prepend chunk-situating context before embedding) as a default quality lever.
    - **GraphRAG** (entity/relationship graphs over the corpus) is mainstream for multi-hop and global-summary questions where flat top-k retrieval underperforms.
    - Late-interaction / multi-vector retrieval (ColBERT, and **ColPali** for visually-rich documents) is a production option for high-recall and document-image retrieval. Evaluate RAG with structured metrics (faithfulness, context precision/recall) via **RAGAS**.

    **Sources:** [Anthropic — Contextual Retrieval](https://www.anthropic.com/news/contextual-retrieval) · [Microsoft — GraphRAG](https://microsoft.github.io/graphrag/) · [ColPali — efficient document retrieval with vision-language models (paper)](https://arxiv.org/abs/2407.01449) · [RAGAS — RAG evaluation framework](https://docs.ragas.io/)
